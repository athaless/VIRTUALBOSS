VirtualBOSS.modulo_PONTO_FOLGA
O modulo_PONTO_FOLGA é uma ferramenta do gestor de RH, que é utilizada para registrar as folgas concedidas bem como os afastamentos do trabalho dos colaboradores (que possuam documento atestando ou justificando a falta aceito pela empresa) bem como suas férias e licenças (ex: licença paternidade e maternidade). O módulo também possui flexibilidade para registrar outros eventos, de acordo com as necessidades da empresa.
Visualização dos afastamentos no espelho de ponto
Todos os eventos de folga e afastamento que são gravados no modulo_PONTO_FOLGA são também visualizados no espelho do ponto do colaborador, servindo para manter o histórico destes eventos, dando ao gestor de RH a possibilidade de organizar o rodízio de férias de seus colaboradores, por exemplo.

Conceito de Aplicação/Módulo

Considerando a palavra "Aplicação" como sinônima de "Módulo": onde determinado grupo de ações sobre uma entidade, conjunto de tabelas ou qualquer abstração de cadastros pode ser realizada, temos neste sistema diversos módulos acessíveis aos usuários cadastrados no sistema, relacionando estas ações com permissões de operação e acesso as mesmas que podem ser configuradas neste nesta aplicação corrente, para cada usuário.

Cada módulo possui um conjunto de ações que são passíveis de se registrar um direito de execução sobre ela para cada usuário cadastrado, onde podemos resumir o conceito em Ações e Direitos, onde:

"ação" é uma operação da aplicação e "direito" é a permissão de executar uma determinada ação.


A aplicação e suas ações

Uma aplicação (módulo) possui operações que são passíveis de controle de permissões, exemplo: este módulo de USUÁRIOS possui as operações de INS, UPD, DEL e VIEW (Inserção, Alteração, Deleção e Visualização). Desta forma, para cada conta de usuário, é possível definir quais operações ele pode ou não realizar neste ou em qualquer outro módulo determinado.

O usuário e seus direitos

Cada usuário tem ou não a ação relativa a aplicação, ou seja, uma vez que o módulo de usuários tem estas operações INS, UPD, DEL e VIEW, cada usuário deverá ter ou não permissão em cada uma delas dentro desta aplicação. Exemplo:

USER_ID......: fulano
APLICAÇÃO.: modulo_usuario (operações: INS, UPD, DEL, COPY, UPD_DIR e VIEW)
DIREITOS....: INS=TRUE//UPD=TRUE//DEL=FALSE//VIEW=TRUE
CAMPOS, "REGRA de NEGÓCIO" e FEATURES

Dentre os campos existentes para o cadastro ao qual este módulo trabalha, destacamos abaixo apenas os de especial relevância, com relação a alguma "regra de negócio" ao qual tal campo se relacione, a saber:

ID Usuário: ID - identificador único do usuário em todo sistema (até 50 caracteres)
Dt Ini: Data de início do afastamento
Dt Fim:Data do final do afastamento
Categoria: tipo de afastamento (folga, licença, férias, etc)


Regras de negócios

Este módulo se relaciona com o espelho de ponto do usuário, que exibe os eventos de afastamento do colaborador do período.



Features

Além das funções básicas de qualquer módulo do sistema - Inserir, Alterar, Remover e Pesquisar (consideramos um módulo como composto de forma abstrata sempre por estes elementos: dialog(s), filtro(s) e grade de dados), podemos ter features extras conforme o caso.
